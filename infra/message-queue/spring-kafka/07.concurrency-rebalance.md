# 7. 컨테이너 동시성과 리밸런싱

## 동시성 — concurrency

- `ConcurrentMessageListenerContainer`가 `concurrency` 수만큼 `KafkaMessageListenerContainer`를 만든다 — 각각 **독립 스레드 + 컨슈머**
- 설정 방법
    ```java
    factory.setConcurrency(3);                       // 팩토리 단위
    // 또는 리스너 단위: @KafkaListener(concurrency = "3")
    ```
- 동시성 ↔ 파티션 수 관계
    - `concurrency ≤ 파티션 수`: 파티션이 컨슈머들에 분배되어 병렬 처리
    - `concurrency > 파티션 수`: **초과 컨슈머는 idle** (할당받을 파티션이 없음) → 파티션 수가 실질 병렬도의 상한
    - 예: 토픽 3개 × 5파티션에 `concurrency=15` → 활성 5개(각 토픽 1파티션씩), 나머지 10개 idle
- 순서: **한 파티션은 한 컨테이너(스레드)가 전담** → 파티션 내 순서 보장, 파티션 간에는 병렬

## 파티션 할당 전략

- 기본 `RangeAssignor` — 토픽별로 연속 파티션을 할당해, 여러 토픽 구독 시 분배가 불균등할 수 있음
- 대안: `RoundRobinAssignor`(균등 분배) / `CooperativeStickyAssignor`(협력적·점진적 리밸런싱)
    ```yaml
    spring:
      kafka:
        consumer:
          properties:
            "[partition.assignment.strategy]": org.apache.kafka.clients.consumer.RoundRobinAssignor
    ```
- 할당 전략 이론은 [6.4 컨슈머 파티션 할당 전략](../practical-kafka/06.kafka-consumer/6.4.consumer-partition-assign.md) 참고

## 리밸런싱 — ConsumerAwareRebalanceListener

- 리밸런싱: 컨슈머 합류/이탈 시 그룹 코디네이터가 파티션을 재할당 (이론: [6.2 그룹 코디네이터](../practical-kafka/06.kafka-consumer/6.2.group-coordinator.md))
- spring-kafka는 카프카의 `ConsumerRebalanceListener`를 확장한 `ConsumerAwareRebalanceListener`(컨슈머 접근 가능)를 제공 — 4개 콜백

| 콜백 | 시점 / 용도 |
|---|---|
| `onPartitionsRevokedBeforeCommit` | 파티션 회수 직전 — 대기 중인 수동 ack 처리 |
| `onPartitionsRevokedAfterCommit` | 보류 오프셋 커밋 후 — 외부 저장소에 오프셋 저장 |
| `onPartitionsAssigned` | 파티션 할당 시 — 특정 오프셋으로 `seek` |
| `onPartitionsLost` | 파티션 상실 시 (2.4+) |

- 등록: `containerProperties.setConsumerRebalanceListener(...)`
    ```java
    cp.setConsumerRebalanceListener(new ConsumerAwareRebalanceListener() {
        public void onPartitionsRevokedAfterCommit(Consumer<?,?> c, Collection<TopicPartition> parts) {
            store(c.position(tp));               // 외부 저장소에 현재 오프셋 저장
        }
        public void onPartitionsAssigned(Consumer<?,?> c, Collection<TopicPartition> parts) {
            c.seek(tp, tracker.getOffset() + 1); // 저장해둔 위치로 복원
        }
    });
    ```
- 잦은 리밸런싱을 줄이려면 스태틱 멤버십([6.3](../practical-kafka/06.kafka-consumer/6.3.static-membership.md)) 활용
- KIP-848 새 리밸런스 프로토콜(4.0+) 지원 — 점진적 리밸런싱이라 `onPartitionsAssigned`가 더 작은 단위로 여러 번 호출될 수 있음
