# 3. @KafkaListener로 메시지 소비

## @KafkaListener 동작 방식

- `@KafkaListener`를 붙인 빈 메서드가 곧 컨슈머 — `KafkaConsumer.poll()` 루프를 직접 작성하지 않음
- 호출 흐름
    - `@EnableKafka`가 있어야 활성화 (Boot가 자동 구성)
    - 리스너 컨테이너 팩토리(기본 `kafkaListenerContainerFactory`)가 메서드마다 컨테이너 생성
    - 컨테이너가 poll → 메서드를 `MessagingMessageListenerAdapter`로 감싸 역직렬화·헤더 변환 후 호출
- Boot 환경이면 `@EnableKafka`와 기본 팩토리가 자동 구성되므로, 보통 메서드에 애너테이션만 붙이면 됨

## 리스너 컨테이너

- `@KafkaListener` 뒤에서 실제 폴링·스레드·메서드 호출을 담당하는 인프라
- 두 구현체
    - `KafkaMessageListenerContainer` — 단일 스레드, 모든 토픽/파티션을 한 스레드에서 수신
    - `ConcurrentMessageListenerContainer` — `concurrency` 수만큼 `KafkaMessageListenerContainer`에 위임 → 멀티 스레드 (동시성·파티션 분배는 7단계)
- `@KafkaListener`는 `ConcurrentKafkaListenerContainerFactory`로 컨테이너를 만든다 — Boot가 `kafkaListenerContainerFactory` 이름으로 기본 팩토리를 자동 등록
- 비-Boot거나 커스터마이즈가 필요하면 팩토리를 직접 구성
    ```java
    @Configuration
    @EnableKafka
    class KafkaConfig {
        @Bean
        ConcurrentKafkaListenerContainerFactory<Integer, String> kafkaListenerContainerFactory() {
            var factory = new ConcurrentKafkaListenerContainerFactory<Integer, String>();
            factory.setConsumerFactory(consumerFactory());  // ConsumerFactory 주입 (역직렬화 설정은 4단계)
            factory.setConcurrency(3);
            return factory;
        }
    }
    ```

## 주요 속성

| 속성 | 설명 |
|---|---|
| `id` | 리스너 식별자. `idIsGroup=true`(기본)면 `group.id`로도 사용 |
| `groupId` | 컨슈머 그룹을 명시적으로 지정 (`id`-as-group보다 우선) |
| `topics` | 구독할 토픽 이름 (복수 가능) |
| `topicPattern` | 정규식으로 토픽 구독 |
| `topicPartitions` | 특정 파티션만 명시적으로 할당 (`@TopicPartition`) |
| `containerFactory` | 사용할 팩토리 빈 이름 (기본값 외 팩토리 지정 시) |
| `concurrency` | 동시 컨슈머 수 (→ 7단계) |
| `batch` | 배치 리스너 활성화 |
| `autoStartup` | 기동 시 자동 시작 여부 (기본 true) |
| `properties` | 컨슈머 옵션 개별 오버라이드 (단 `group.id`·`client.id`는 불가 → `groupId`·`clientIdPrefix` 사용) |

- 모든 속성은 SpEL(`#{…}`)·프로퍼티 플레이스홀더(`${…}`) 지원 — 예: `topics = "${app.topic}"`

## 메서드 시그니처 — 무엇을 주입받나

- 필요한 것만 파라미터로 선언하면 어댑터가 맞춰서 주입
    ```java
    // 1) 페이로드 직접 — 역직렬화된 값만
    @KafkaListener(id = "g1", topics = "myTopic")
    public void listen(String data) { }

    // (이하 메서드도 @KafkaListener 필요, 생략) 2) ConsumerRecord — 키·값·메타데이터 전부 접근
    public void listen(ConsumerRecord<Integer, String> record) { }  // record.key()/value()/offset()/partition()

    // 3) @Payload + @Header — 페이로드와 헤더를 분리 주입
    public void listen(@Payload String data,
            @Header(KafkaHeaders.RECEIVED_KEY) Integer key,
            @Header(KafkaHeaders.RECEIVED_TOPIC) String topic,
            @Header(KafkaHeaders.RECEIVED_PARTITION) int partition,
            @Header(KafkaHeaders.OFFSET) long offset) { }

    // 4) Acknowledgment(수동 커밋 → 6단계), Consumer(컨슈머 직접 접근)
    public void listen(String data, Acknowledgment ack, Consumer<?, ?> consumer) { }
    ```
- 페이로드+메타데이터를 한 번에 받으려면 `ConsumerRecordMetadata` 파라미터도 가능

## 토픽·파티션 지정

```java
@KafkaListener(topics = {"t1", "t2"})                  // 토픽 이름(복수)
@KafkaListener(topicPattern = "my.*")                  // 정규식 패턴 구독
@KafkaListener(topicPartitions = @TopicPartition(      // 특정 파티션 + 초기 오프셋
    topic = "t1", partitions = "0-5, 7",               // 범위 표기 가능
    partitionOffsets = @PartitionOffset(partition = "*", initialOffset = "0")))
```

## 배치 리스너

- 한 번의 `poll()` 결과를 묶어서 한 번에 처리
    ```java
    factory.setBatchListener(true);                    // 팩토리 설정 (또는 @KafkaListener(batch = true))

    @KafkaListener(id = "b", topics = "t", containerFactory = "batchFactory")
    public void listen(List<String> payloads) { }      // List<ConsumerRecord<..>>, ConsumerRecords<?,?>도 가능
    ```

## 클래스 레벨 @KafkaListener + @KafkaHandler

- 한 토픽에 여러 타입의 메시지가 섞일 때, 페이로드 타입별로 메서드 분기
    ```java
    @KafkaListener(id = "multi", topics = "t")         // 클래스에 부착
    static class Listener {
        @KafkaHandler public void handle(String s) { }
        @KafkaHandler public void handle(Order o) { }
        @KafkaHandler(isDefault = true) public void handle(Object o) { }  // 매칭 없을 때 기본 핸들러
    }
    ```

## @SendTo로 응답 전송

- 리스너의 리턴값을 다른 토픽으로 발행 (요청-응답 패턴)
    ```java
    @KafkaListener(id = "f", topics = "inTopic")
    @SendTo("outTopic")                                // 리턴값을 outTopic으로 발행
    public String listen(String data) { return data.toUpperCase(); }
    ```
