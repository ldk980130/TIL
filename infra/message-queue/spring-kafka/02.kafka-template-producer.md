# 2. KafkaTemplate으로 메시지 발행

## KafkaTemplate과 ProducerFactory

- 발행은 두 빈이 협력
    - `ProducerFactory` — 카프카 `Producer` 인스턴스를 생성 (구현체 `DefaultKafkaProducerFactory`)
    - `KafkaTemplate` — `ProducerFactory`가 만든 producer로 실제 발행을 수행하는 템플릿
- Boot 자동 구성이면 둘 다 자동 등록되므로([1단계](01.spring-kafka-intro.md)) 보통 `KafkaTemplate`만 주입해서 사용
- 직접 구성하는 경우 (비-Boot 등)
    ```java
    // ProducerFactory: 설정 맵으로 producer 생성 (key/value 직렬화는 4단계)
    var pf = new DefaultKafkaProducerFactory<String, String>(Map.of(
        ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092",
        ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class,
        ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class));
    var template = new KafkaTemplate<>(pf);
    ```
- producer 공유 방식
    - 기본: **단일 producer를 모든 스레드가 공유** (`KafkaProducer`는 thread-safe, 카프카 권장 방식)
    - `producerPerThread = true`: 스레드별 producer 생성 → 사용 후 `closeThreadBoundProducer()`로 직접 정리 필요

## send() 메서드 변형

- 모든 `send()` / `sendDefault()`는 `CompletableFuture<SendResult<K, V>>`를 반환 (비동기)
    ```java
    // 토픽 명시
    template.send("topic", value);
    template.send("topic", key, value);
    template.send("topic", partition, key, value);
    template.send("topic", partition, timestamp, key, value);
    template.send(producerRecord);   // ProducerRecord 직접 구성
    template.send(message);          // spring-messaging Message<?> (아래 참고)

    // 기본 토픽 사용: setDefaultTopic()으로 지정 후 토픽 생략
    template.setDefaultTopic("topic");
    template.sendDefault(key, value);
    ```
- 파티션·키를 줄수록 카프카의 파티셔닝/정렬 동작을 직접 제어 (이론: [5.1 파티셔너](../practical-kafka/05.kafka-producer/5.1.partioner.md))

## 발행 결과 처리 — 비동기 vs 동기

- `send()`는 즉시 반환하고 전송은 백그라운드에서 진행 → 결과는 future로 처리
- `SendResult<K, V>` 구성
    - `ProducerRecord` — 보낸 레코드 원본
    - `RecordMetadata` — 저장된 위치 정보 (topic, partition, offset, timestamp)
- 비동기 — 콜백 (논블로킹, 권장)
    ```java
    template.send("topic", data).whenComplete((result, ex) -> {
        if (ex == null) result.getRecordMetadata().offset();  // 성공: 오프셋 등 확인
        else /* 실패 처리 */;
    });
    ```
- 동기 — `get()`으로 블로킹 (처리량↓, 발행 확정·순서 보장이 필요할 때만)
    ```java
    try {
        var result = template.send("topic", data).get(10, TimeUnit.SECONDS);
    } catch (ExecutionException e) {
        // e.getCause()는 KafkaProducerException (실패한 producerRecord 보유)
    } // TimeoutException, InterruptedException도 처리
    ```

## 배치 전송과 flush

- `send()` 호출이 곧 네트워크 전송은 아님 — `linger.ms` / `batch.size`에 따라 모아서 배치 전송 (이론: [5.2 프로듀서 배치](../practical-kafka/05.kafka-producer/5.2.producer-batch.md))
- `flush()` — 대기 중인 레코드를 즉시 전송 (`linger.ms`를 둔 상태에서 부분 배치를 바로 보내고 싶을 때)
- `new KafkaTemplate<>(pf, true)` — autoFlush=true면 매 send마다 flush (처리량 저하, 특수 상황에만)

## Message<?> 추상화로 발행

- spring-messaging의 `Message<?>`로 페이로드 + 헤더를 함께 구성해 발행
    ```java
    Message<String> msg = MessageBuilder.withPayload("data")
        .setHeader(KafkaHeaders.TOPIC, "myTopic")
        .setHeader(KafkaHeaders.KEY, "myKey")
        .setHeader(KafkaHeaders.PARTITION, 0)
        .build();
    template.send(msg);   // 토픽/키/파티션을 인자 대신 헤더로 지정
    ```
- 지정 가능한 헤더: `KafkaHeaders.TOPIC`, `KEY`, `PARTITION`, `TIMESTAMP`

## ProducerListener — 전역 발행 콜백

- 개별 future 대신, `KafkaTemplate`의 모든 발행 결과를 한곳에서 받는 전역 리스너
    ```java
    template.setProducerListener(new ProducerListener<>() {
        public void onSuccess(ProducerRecord<K,V> rec, RecordMetadata meta) { ... }
        public void onError(ProducerRecord<K,V> rec, RecordMetadata meta, Exception ex) { ... }
    });
    ```
- 기본값은 `LoggingProducerListener` (에러만 로깅)
- 건별 처리는 `send().whenComplete(...)`, 공통 로깅·메트릭 같은 횡단 처리는 `ProducerListener`가 적합
