# 1. Spring Kafka 개요와 환경 설정

## Spring for Apache Kafka란

- 카프카 네이티브 Java 클라이언트(`org.apache.kafka.clients`)를 Spring 스타일로 추상화한 프로젝트
- 네이티브 클라이언트의 `KafkaProducer.send()` 호출이나 `KafkaConsumer.poll()` 루프를 직접 작성하는 대신, 선언적으로 발행·소비 가능
- 핵심 구성 요소
    - `KafkaTemplate` — 메시지 발행용 템플릿 (`JdbcTemplate`과 유사한 추상화)
    - `@KafkaListener` — 애너테이션 기반 컨슈머 메서드 정의
    - 리스너 컨테이너 — `ConcurrentKafkaListenerContainerFactory` 등, 컨슈머 폴링 루프·스레드·리밸런싱을 대신 관리하는 인프라
    - 프로듀서/컨슈머 팩토리 — `DefaultKafkaProducerFactory`, `DefaultKafkaConsumerFactory`
    - 그 외 토픽 관리(`KafkaAdmin`), 직렬화/역직렬화, 트랜잭션, 에러 핸들링, 모니터링 지원

## 의존성

- 환경에 따라 스타터 또는 `spring-kafka`를 추가 (둘은 택일)
    - Spring Boot 4.x — 전용 스타터 (버전은 Boot BOM이 관리하므로 생략)
    - Spring Boot 3.x 이하 / 비-Boot — `spring-kafka` 직접 추가 (Boot면 BOM이 버전 관리, 비-Boot면 버전 명시)
    - 테스트(EmbeddedKafka 등)는 `spring-kafka-test` 추가 (9단계에서 다룸)
    ```gradle
    // Boot 4.x: 전용 스타터
    implementation 'org.springframework.boot:spring-boot-starter-kafka'
    // Boot 3.x 이하 / 비-Boot: 스타터 대신 직접 추가 (비-Boot면 ':4.1.0' 처럼 버전 명시)
    implementation 'org.springframework.kafka:spring-kafka'
    // 테스트
    testImplementation 'org.springframework.kafka:spring-kafka-test'
    ```

## 버전 호환성

| spring-kafka | kafka-clients | Spring Framework | 최소 Java |
|---|---|---|---|
| 4.1.x | 4.0.x | 7.0.x | 17 |

- Boot를 쓰면 위 버전들을 Boot BOM이 맞춰주므로, 직접 버전을 명시하지 않는 것이 안전
- kafka-clients는 브로커와 양방향 호환되어 클라이언트/브로커 버전이 정확히 같을 필요는 없음

## 자동 구성 (Spring Boot)

- `spring.kafka.*` 프로퍼티만 있으면 Boot가 아래 빈들을 자동 등록 → 바로 주입해서 사용
    - `ProducerFactory` / `ConsumerFactory` — 프로듀서·컨슈머 인스턴스 생성 팩토리
    - `KafkaTemplate` — 발행용 (`ProducerFactory` 기반)
    - `KafkaListenerContainerFactory` — `@KafkaListener`가 사용하는 기본 컨테이너 팩토리
    - `KafkaAdmin` — 관리 작업용. `NewTopic` 빈을 선언하면 앱 기동 시 토픽 자동 생성 (이미 있으면 무시)
    - `KafkaTransactionManager` — `spring.kafka.producer.transaction-id-prefix`가 설정된 경우에만
    - 메시지 컨버터(`RecordMessageConverter` 등)도 자동 구성

## application.yml ↔ 네이티브 클라이언트 옵션 매핑

- Boot 프로퍼티(kebab-case)는 카프카 점(.) 표기 옵션으로 그대로 매핑됨 (아래 주석이 매핑 대상)
    ```yaml
    spring:
      kafka:
        bootstrap-servers: localhost:9092      # bootstrap.servers (모든 클라이언트 공통)
        producer:
          acks: all                            # acks
          retries: 3                           # retries
          batch-size: 16384                    # batch.size
          compression-type: snappy             # compression.type
          key-serializer: ...StringSerializer  # key.serializer
          value-serializer: ...StringSerializer# value.serializer
        consumer:
          group-id: my-group                   # group.id
          auto-offset-reset: earliest          # auto.offset.reset
          enable-auto-commit: false            # enable.auto.commit
          max-poll-records: 500                # max.poll.records
          key-deserializer: ...StringDeserializer  # key.deserializer
          value-deserializer: ...StringDeserializer# value.deserializer
    ```
- 각 옵션의 의미는 기존 [3.3 프로듀서 옵션](../practical-kafka/03.kafka-basic-structure/3.3.kafka-producer-basic.md) · [3.4 컨슈머 옵션](../practical-kafka/03.kafka-basic-structure/3.4.kafka-consumer.basic.md) 노트와 동일
- prefix별 적용 대상
    - `spring.kafka.properties.*` — 모든 클라이언트 공통
    - `spring.kafka.producer.*` / `consumer.*` / `admin.*` / `streams.*` — 해당 클라이언트에만 적용 (공통 설정을 덮어씀)

## 노출되지 않은 옵션 주입

- `KafkaProperties`가 직접 노출하는 건 주로 중요도 HIGH 옵션 일부 + 일부 MEDIUM/LOW
- 노출되지 않은 옵션은 `properties` 네임스페이스에 raw 키를 그대로 전달 (키에 점이 있으므로 `"[...]"`로 감쌈)
    ```yaml
    spring:
      kafka:
        consumer:
          properties:
            "[max.partition.fetch.bytes]": 1048576  # KafkaProperties에 없는 옵션을 raw 키로 전달
    ```
- `spring.kafka.properties.*` 값은 Boot의 명시적 설정보다 우선 적용됨

## 최소 동작 예제

```java
// 발행 — 주입받은 KafkaTemplate 사용
kafkaTemplate.send("topic1", "hello");   // (topic, value)

// 소비 — 메서드에 애너테이션만 붙이면 컨테이너가 폴링·역직렬화 후 호출
@KafkaListener(id = "myId", topics = "topic1")
public void listen(String in) { ... }    // 역직렬화된 value가 파라미터로 주입
```
