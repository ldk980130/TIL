# 9. EmbeddedKafka 테스트

## spring-kafka-test 의존성

- 내장 브로커(`EmbeddedKafkaBroker`)와 테스트 유틸 제공
    ```gradle
    // Boot 4.x: 전용 테스트 스타터
    testImplementation 'org.springframework.boot:spring-boot-starter-kafka-test'
    // Boot 3.x 이하 / 비-Boot: 직접 추가
    testImplementation 'org.springframework.kafka:spring-kafka-test'
    ```
- 카프카 4.0+는 KRaft 전용 — 내장 브로커도 ZooKeeper 없이 `EmbeddedKafkaKraftBroker`로 동작

## @EmbeddedKafka — 내장 브로커 기동

- 테스트 클래스/메서드에 붙이면 인메모리 카프카 브로커를 띄움 → 실제 브로커 없이 통합 테스트
- 주요 속성

| 속성 | 용도 |
|---|---|
| `topics` | 생성할 토픽 |
| `partitions` | 토픽당 파티션 수 (기본 1) |
| `count` | 브로커 개수 |
| `ports` | 브로커 포트 (0=랜덤) |
| `brokerProperties` | 브로커 설정 오버라이드 |
| `bootstrapServersProperty` | 브로커 주소를 노출할 프로퍼티명 (기본 `spring.kafka.bootstrap-servers`, 3.0.10+) |

- `bootstrapServersProperty`가 기본값이라 **Boot 자동 구성이 내장 브로커 주소를 자동 인식** → 별도 설정 없이 `KafkaTemplate`·`@KafkaListener`가 내장 브로커에 연결

## @SpringBootTest와 함께

```java
@SpringBootTest
@EmbeddedKafka(topics = "test-topic", partitions = 1)
class MyKafkaTest {
    @Autowired KafkaTemplate<String, String> template;       // 내장 브로커에 연결됨
    @Autowired EmbeddedKafkaBroker broker;
}
```

## KafkaTestUtils — 컨슈머/레코드 헬퍼

- 테스트용 컨슈머·프로듀서 설정과 레코드 수신을 간편화
    - `consumerProps(group, autoCommit, broker)` / `producerProps(broker)` — 설정 맵 생성
    - `getSingleRecord(consumer, topic)` — 레코드 1건 대기 후 반환
    - `getRecords(consumer)` — 받은 레코드 전부

## ContainerTestUtils.waitForAssignment

- 리스너가 **파티션을 할당받기 전에 발행하면 메시지를 놓칠 수 있음** → 할당 완료까지 대기 후 발행
    ```java
    ContainerTestUtils.waitForAssignment(container, broker.getPartitionsPerTopic());
    ```

## 단언 헬퍼

- AssertJ `KafkaConditions`: `key(k)`, `value(v)`, `keyValue(k, v)`, `partition(p)`
- Hamcrest `KafkaMatchers`: `hasKey(k)`, `hasValue(v)`, `hasPartition(p)`

## 최소 E2E 예제

```java
@EmbeddedKafka(topics = "test-topic", partitions = 1)
class KafkaTemplateTest {
    @Autowired EmbeddedKafkaBroker broker;

    @Test
    void sendAndReceive() {
        // 컨슈머 준비 후 토픽 구독
        var consumer = new DefaultKafkaConsumerFactory<>(
            KafkaTestUtils.consumerProps("g", "false", broker)).createConsumer();
        broker.consumeFromAnEmbeddedTopic(consumer, "test-topic");

        // 발행
        var template = new KafkaTemplate<>(new DefaultKafkaProducerFactory<>(
            KafkaTestUtils.producerProps(broker)));
        template.send("test-topic", "hello");

        // 수신·검증
        var received = KafkaTestUtils.getSingleRecord(consumer, "test-topic");
        assertThat(received).has(value("hello"));   // KafkaConditions.value
    }
}
```

## 주의점

- 테스트마다 **다른 토픽** 사용 (테스트 간 메시지 간섭 방지), 필요 시 `@DirtiesContext`
- 브로커 없이 단위 테스트하려면 `MockProducerFactory` / `MockConsumerFactory` (3.0.7+)
