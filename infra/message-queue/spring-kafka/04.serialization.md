# 4. 직렬화 / 역직렬화

## 기본 구조

- 프로듀서는 key/value `Serializer`, 컨슈머는 key/value `Deserializer`로 객체 ↔ byte[] 변환
- 설정 위치
    - `ProducerFactory`/`ConsumerFactory`에 클래스·인스턴스·`Supplier`로 주입
    - 또는 `spring.kafka.producer.value-serializer` 등 프로퍼티로 ([1단계](01.spring-kafka-intro.md) 매핑)
- 카프카 기본 제공: `StringSerializer`, `IntegerSerializer`, `ByteArraySerializer` 등 (`org.apache.kafka.common.serialization`)

## JSON 직렬화

- spring-kafka가 제공하는 JSON 직렬화기 (`org.springframework.kafka.support.serializer`)
    - `JacksonJsonSerializer<T>` — 객체를 JSON `byte[]`로
    - `JacksonJsonDeserializer<T>` — 생성자에 `targetType` 필요, JSON을 객체로
    - 내부 `ObjectMapper`는 커스터마이즈 가능
- 버전 주의: spring-kafka 3.x에선 클래스명이 `JsonSerializer`/`JsonDeserializer` (4.x에서 `Jackson` 접두사로 개명, 프로퍼티 `spring.json.*`는 동일)
    ```java
    var deserializer = new JacksonJsonDeserializer<>(Thing.class);  // targetType 지정
    ```

## 타입 정보 처리

- 기본적으로 직렬화 시 `__TypeId__` 헤더에 타입을 실어 보냄 → 역직렬화 시 그 타입으로 복원 (다형성 지원)
- 주요 설정 (Java 상수 / Boot 프로퍼티)

| 설정 (상수 / 프로퍼티) | 용도 |
|---|---|
| `VALUE_DEFAULT_TYPE` / `spring.json.value.default.type` | 헤더가 없을 때 사용할 기본 타입 |
| `TRUSTED_PACKAGES` / `spring.json.trusted.packages` | 역직렬화 허용 패키지. 기본 `java.util`·`java.lang`, `*`는 전체 허용 |
| `TYPE_MAPPINGS` / `spring.json.type.mapping` | `토큰:FQCN` 매핑. 시스템 간 클래스명이 다를 때 |
| `USE_TYPE_INFO_HEADERS` / `ignoreTypeHeaders()` | 보낸 타입 헤더를 무시하고 기본 타입으로 강제 |

```yaml
spring:
  kafka:
    producer:
      value-serializer: org.springframework.kafka.support.serializer.JacksonJsonSerializer
    consumer:
      value-deserializer: org.springframework.kafka.support.serializer.JacksonJsonDeserializer
      properties:
        "[spring.json.value.default.type]": com.example.MyValue
        "[spring.json.trusted.packages]": com.example   # 신뢰 패키지 제한 (보안)
```

- 보안: `TRUSTED_PACKAGES`를 좁게 두면 신뢰하지 않는 패키지의 클래스 역직렬화를 차단 — `*`는 편하지만 임의 클래스 역직렬화 위험

## ErrorHandlingDeserializer

- 역직렬화 실패가 **컨테이너를 멈추지 않게** 하는 위임 래퍼
- 동작
    - 실제 deserializer에 위임하다 예외가 나면, 던지지 않고 예외를 헤더에 담아 흐름을 통과시킴
    - 이후 컨테이너의 에러 핸들러가 실패 레코드를 처리 (→ 5단계)
    ```java
    // 컨슈머의 deserializer를 래퍼로 지정
    props.put(VALUE_DESERIALIZER_CLASS_CONFIG, ErrorHandlingDeserializer.class);
    // 래퍼가 실제로 위임할 deserializer
    props.put(ErrorHandlingDeserializer.VALUE_DESERIALIZER_CLASS, JacksonJsonDeserializer.class);
    ```
- `VALUE_FUNCTION`에 `Function<FailedDeserializationInfo, T>`를 주면 실패 시 fallback 객체 생성도 가능

## 토픽·타입별 다른 직렬화기

- 한 토픽/팩토리에서 상황에 따라 다른 직렬화기를 쓰는 위임 직렬화기들
    - `DelegatingByTypeSerializer` — Java 타입별 (`Map<Class, Serializer>`)
    - `DelegatingByTopicSerializer` / `DelegatingByTopicDeserializer` — 토픽(정규식)별
    - `DelegatingSerializer` / `DelegatingDeserializer` — 헤더 셀렉터로 선택
    ```java
    new DelegatingByTypeSerializer(Map.of(
        byte[].class, new ByteArraySerializer(),
        String.class, new StringSerializer()));
    ```

## 스키마 레지스트리 연동

- Avro 등 스키마 기반 직렬화는 Confluent의 serializer를 `ProducerFactory`/`ConsumerFactory`에 설정해 사용
- 스키마 개념·호환성 규칙은 기존 정리 참조 — [10.2 카프카와 스키마 레지스트리](../practical-kafka/10.schema-registry/10.2.kafka-registry.md), [10.4 호환성](../practical-kafka/10.schema-registry/10.4.schema-compatibility.md)
